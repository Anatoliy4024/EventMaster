
# EventMaster Project

## Версия от 30.08.2024

### 1. Суть проекта
Проект **EventMaster** предназначен для автоматизации процессов организации и управления мероприятиями. В рамках проекта реализованы два Telegram бота:

- **AdminBot**: предназначен для администрирования и управления мероприятиями.
- **PicnicBot**: бот для пользователей, который позволяет им бронировать мероприятия.

### 2. Файловая структура

Проект разделен на два основных компонента:

1. **AdminBot**: Основной бот для администрирования мероприятий.
2. **PicnicBot**: Бот для взаимодействия с конечными пользователями, которые хотят забронировать мероприятие.

### 3. Архитектура

#### 3.1 Принцип модульности

Мы придерживаемся принципа модульности, разделяя функциональность на независимые модули. Это позволяет легко расширять и поддерживать код, а также повторно использовать его в других проектах.

#### 3.2 Модули AdminBot

1. **helpers/database_helpers.py**: Вспомогательные функции для работы с базой данных и отправки проформ пользователям.
2. **keyboards/**: Создание интерактивных клавиатур для взаимодействия с пользователем.
3. **scenarios/**:
   - `user_scenario.py`: Сценарий взаимодействия с пользователем.
   - `organizer_scenario.py`: Сценарий взаимодействия с организатором.
   - `service_scenario.py`: Сценарий взаимодействия с сервисной службой.
4. **main.py**: Основной файл для запуска AdminBot.

#### 3.3 Модули PicnicBot

1. **step_handlers/**: Обработчики шагов выбора (язык, дата, время, количество персон, стиль мероприятия).
2. **helpers/**:
   - `db_helpers.py`: Работа с базой данных.
   - `picnic_functions.py`: Специфичные функции для PicnicBot.
3. **keyboards/**: Клавиатуры и кнопки для взаимодействия с пользователями.
4. **db/**: Работа с базой данных и файл SQLite.
5. **payments/**: Модули для обработки платежей и генерации проформ.
6. **main.py**: Основной файл для запуска PicnicBot.

#### 3.4 Общие модули (shared/)

- **config.py**: Конфигурационные данные для обоих ботов.
- **constants.py**: Константы и временные данные.
- **translations.py**: Переводы текстов сообщений и кнопок на различные языки.
- **helpers.py**: Вспомогательные функции, используемые обоими ботами.

### 4. Новые компоненты в версии 2.0

#### 4.1 Добавлены следующие модули

1. **PicnicBot**: Новый бот, интегрированный с EventMaster, который позволяет пользователям бронировать мероприятия.
2. **step_handlers/**: Модули для пошаговой обработки взаимодействия с пользователем.
3. **payments/**: Модули для обработки платежей и генерации проформ.

### 5. Примечания

- В версии 2.0 реализована интеграция PicnicBot с AdminBot, что позволяет использовать общие ресурсы и модули.
- Улучшена структура проекта, что облегчает его дальнейшую поддержку и развитие.

# описание файлов

1. ** Описание файла db_helpers.py
Файл: bot/picnic_bot/helpers/db_helpers.py

Назначение: Этот файл содержит функции для работы с базой данных SQLite, используемой в PicnicBot. Основная цель — управление записями в таблице заказов (orders), включая создание новых записей, обновление существующих данных, а также проверку существования записей. Логирование всех операций помогает отслеживать действия и возможные ошибки при взаимодействии с базой данных.

Ключевые компоненты:

Импорты:

sqlite3: Модуль для работы с базой данных SQLite.
logging: Модуль для ведения логов (записей) операций и ошибок.
create_connection: Функция, импортированная из shared/helpers.py, которая устанавливает соединение с базой данных SQLite.
DATABASE_PATH: Константа из shared/config.py, содержащая путь к файлу базы данных.
Функция picnic_db_operations:

Описание: Эта функция выполняет различные операции с базой данных в PicnicBot, включая проверку существования записи, обновление записи или создание новой записи в таблице orders.
Параметры:
user_id: Идентификатор пользователя Telegram, используется для связывания записей с конкретным пользователем.
session_number: Номер текущей сессии пользователя, который уникально идентифицирует сессию.
update_data: Необязательный словарь, содержащий данные для обновления в существующей записи.
Логика работы:
Соединение с базой данных: Функция устанавливает соединение с базой данных с помощью create_connection.
Проверка существования записи: SQL-запрос проверяет, существует ли запись с указанными user_id и session_number.
Обновление записи: Если запись существует и переданы данные для обновления, то запись обновляется.
Создание новой записи: Если запись не существует, создается новая запись в таблице orders с базовыми значениями.
Логирование: Все действия и возможные ошибки логируются для последующего анализа.
Закрытие соединения: После выполнения всех операций соединение с базой данных закрывается.
Примечание: Файл db_helpers.py играет важную роль в управлении данными пользователей, обеспечивая надежное и логируемое взаимодействие с базой данных.

Описание файла order_calculation.py
Файл: bot/picnic_bot/helpers/order_calculation.py

Назначение: Этот файл содержит функцию для расчета общей стоимости мероприятия на основе продолжительности и количества участников. Функция использует фиксированные базовые ставки для расчета, что позволяет гибко учитывать дополнительные часы и людей, участвующих в мероприятии.

Ключевые компоненты:

Импорты:

Кодировка: Файл включает директиву # -*- coding: utf-8 -*-, что указывает на использование кодировки UTF-8. Это важно для поддержки русского текста и других символов Unicode в комментариях или строковых значениях.
Функция calculate_total_cost:

Описание: Основная функция, которая рассчитывает общую стоимость мероприятия на основе продолжительности (в часах) и количества участников.
Параметры:
duration: Продолжительность мероприятия в часах.
people_count: Количество участников мероприятия.
Логика работы:
Базовая стоимость: Устанавливается фиксированная базовая стоимость мероприятия в 160 единиц.
Дополнительные часы: Если продолжительность мероприятия превышает 2 часа, за каждый дополнительный час добавляется 30 единиц к стоимости.
Дополнительные участники: Если количество участников превышает 2 человека, за каждого дополнительного человека добавляется 20 единиц к стоимости.
Итоговая стоимость: Общая стоимость рассчитывается как сумма базовой стоимости, стоимости дополнительных часов и стоимости дополнительных участников.
Возвращаемое значение:
total_cost: Итоговая стоимость мероприятия, рассчитанная на основе введенных параметров.
Примечание: Функция calculate_total_cost является простой, но важной частью логики PicnicBot, обеспечивая расчет стоимости мероприятий в зависимости от их продолжительности и количества участников. Она легко модифицируется для включения дополнительных факторов или изменения ставок, что делает ее гибким инструментом для расчета цен.

Описание файла picnic_keyboards.py
Файл: bot/picnic_bot/keyboards/picnic_keyboards.py

Назначение: Этот файл содержит функции для генерации различных клавиатур в боте PicnicBot,
включая календарные клавиатуры, клавиатуры выбора времени, количества участников, стилей мероприятий,
а также клавиатуры выбора языка и подтверждения действий (Да/Нет). 
Клавиатуры создаются динамически и могут адаптироваться к выбранному языку пользователя.

Ключевые компоненты:

Импорты:

InlineKeyboardMarkup и InlineKeyboardButton из библиотеки telegram: используются для создания разметки и кнопок в клавиатурах.
datetime, timedelta и calendar: используются для работы с датами и временем.
logging: используется для логирования, чтобы отслеживать события и ошибки в процессе работы.
Функция to_superscript:

Описание: Преобразует строку цифр в надстрочные символы. Это используется для отображения дней в календаре с надстрочным шрифтом.
Параметры:
num_str: Строка, содержащая цифры.
Возвращаемое значение: Строка с надстрочными цифрами.
Функция generate_month_name:

Описание: Возвращает название месяца на указанном языке.
Параметры:
month: Номер месяца (от 1 до 12).
language: Код языка (например, 'en', 'ru').
Возвращаемое значение: Краткое название месяца на выбранном языке.
Функция generate_calendar_keyboard:

Описание: Генерирует календарную клавиатуру для выбора даты.
Параметры:
month_offset: Смещение месяца относительно текущего (например, 0 — текущий месяц, 1 — следующий).
language: Код языка для отображения названий месяцев и дней недели.
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками для каждого дня месяца и кнопками для переключения между месяцами.
Функция generate_time_selection_keyboard:

Описание: Создает клавиатуру для выбора времени начала и окончания мероприятия.
Параметры:
language: Код языка для отображения текстов на клавиатуре.
stage: Этап выбора времени ('start' или 'end').
start_time: Время начала мероприятия (используется при выборе времени окончания).
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками для выбора времени.
Функция language_selection_keyboard:

Описание: Создает клавиатуру для выбора языка пользователем.
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками для выбора доступных языков.
Функция yes_no_keyboard:

Описание: Создает клавиатуру для подтверждения действия (Да/Нет).
Параметры:
language: Код языка для отображения текстов на клавиатуре.
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками для подтверждения или отмены действия.
Функция generate_person_selection_keyboard:

Описание: Генерирует клавиатуру для выбора количества участников мероприятия.
Параметры:
language: Код языка для отображения текстов на клавиатуре.
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками, отображающими числа от 2 до 21.
Функция generate_party_styles_keyboard:

Описание: Создает клавиатуру для выбора стиля мероприятия.
Параметры:
language: Код языка для отображения текстов на клавиатуре.
Возвращаемое значение: Объект InlineKeyboardMarkup с кнопками для выбора стиля мероприятия.
Примечание: Файл picnic_keyboards.py играет ключевую роль в интерфейсе пользователя, предоставляя удобные и локализованные 
клавиатуры для различных этапов взаимодействия с ботом. Эти клавиатуры помогают пользователю выбрать дату, время,

Назначение: Этот файл служит для хранения различных констант, которые используются в процессе работы бота. В нем содержатся словари с переводами текстов для различных шагов взаимодействия с пользователем, статусы заказов, а также классы для хранения временных данных пользователя и основной информации о пользователе. Ключевые компоненты:

DATABASE_PATH:

Описание: Переменная, которая содержит путь к базе данных SQLite, используемой ботом.
Пример значения:
DATABASE_PATH = os.path.join(os.path.dirname(__file__), 'sqlite.db'): Путь к базе данных относительно директории, в которой находится текущий файл.

Описание: Словарь, содержащий возможные статусы заказов, где каждому статусу соответствует числовое значение. Эти статусы используются для отслеживания этапов обработки заказов.
Пример значений:
"1-не заполнено - невозможно выполнить расчет стоимости": 1
"3-зарезервировано - заказчик оплатил аванс": 3

Описание файла shared/database_logger.py
Этот файл предназначен для управления логированием операций с базой данных SQLite. В нем реализованы функции для записи логов SQL-запросов, сообщений о выполнении операций, а также для выполнения запросов с предварительным логированием.

Компоненты файла:
Импорт библиотек:

logging: Модуль для записи логов.
sqlite3: Модуль для работы с SQLite базой данных.
Настройка логирования:

Логи будут записываться в файл db_operations.log.
Формат логов включает временную метку, уровень сообщения и текст сообщения.
Уровень логирования установлен на INFO, что позволяет логировать сообщения информационного характера и выше.
Указание кодировки UTF-8 гарантирует корректное отображение символов в логе.
Функции:

log_message(message):

Функция для записи произвольного информационного сообщения в лог.
Параметры:
message: Текст сообщения для записи в лог.
Пример использования:
python
Копировать код
log_message('Соединение с базой данных установлено.')
log_query(query, params=()):

Функция для логирования SQL-запроса и его параметров.
Параметры:
query: Текст SQL-запроса.
params: Кортеж с параметрами для SQL-запроса (по умолчанию пустой).
Пример использования:
python
Копировать код
log_query('SELECT * FROM users WHERE id=?', (user_id,))
execute_query_with_logging(conn, query, params=()):

Функция для выполнения SQL-запроса с предварительным логированием запроса и параметров, а также логированием результата (успех или ошибка).
Параметры:
conn: Объект соединения с базой данных SQLite.
query: Текст SQL-запроса.
params: Кортеж с параметрами для SQL-запроса (по умолчанию пустой).
Обработка ошибок:
В случае ошибки при выполнении запроса, ошибка будет зафиксирована в логе.
Пример использования:
python
Копировать код
execute_query_with_logging(conn, 'INSERT INTO users (name, age) VALUES (?, ?)', ('John', 30))
Применение:
Этот модуль может быть использован в проектах, где требуется отслеживать выполнение операций с базой данных для целей отладки и аудита.
Логирование поможет выявить проблемы с выполнением запросов и упростит процесс отладки.

Рекомендации:
Файл лога: Убедитесь, что путь к файлу db_operations.log корректен и доступен для записи в зависимости от окружения,
в котором запускается ваш проект.
Обработка исключений: В текущей реализации обрабатываются только исключения sqlite3.Error. 
Для более полной обработки можно добавить другие виды исключений.

Описание файла shared/helpers.py
Этот файл предоставляет функции для работы с базой данных SQLite, включая создание соединения, выполнение SQL-запросов, а также обработку ошибок и повторные попытки выполнения запросов в случае блокировки базы данных.

Компоненты файла:
Импорт библиотек:

sqlite3: Модуль для работы с базой данных SQLite.
time: Модуль для работы с временными задержками, используемыми при повторных попытках выполнения запросов.
log_message, log_query: Функции логирования из модуля shared.database_logger, используемые для записи логов операций с базой данных.
DATABASE_PATH: Константа, содержащая путь к базе данных, импортированная из файла shared.config.
Функции:

create_connection(db_file):

Создает соединение с базой данных SQLite, указанной в параметре db_file.
Параметры:
db_file: Путь к файлу базы данных SQLite.
Возвращаемое значение:
Объект соединения SQLite или None, если соединение не удалось установить.
Пример использования:
python
Копировать код
conn = create_connection(DATABASE_PATH)
execute_query(conn, query, params=()):

Выполняет SQL-запрос с использованием предоставленного соединения с базой данных.
Параметры:
conn: Объект соединения с базой данных SQLite.
query: Текст SQL-запроса.
params: Кортеж с параметрами для SQL-запроса (по умолчанию пустой).
Возвращаемое значение:
True, если запрос был успешно выполнен, False в случае ошибки.
Особенности:
Логирование запроса и параметров перед выполнением.
Закрытие соединения после выполнения запроса.
Пример использования:
python
Копировать код
execute_query(conn, 'INSERT INTO users (name, age) VALUES (?, ?)', ('John', 30))
execute_query_with_retry(conn, query, params=(), max_retries=5):

Выполняет SQL-запрос с повторными попытками в случае блокировки базы данных.
Параметры:
conn: Объект соединения с базой данных SQLite.
query: Текст SQL-запроса.
params: Кортеж с параметрами для SQL-запроса (по умолчанию пустой).
max_retries: Максимальное количество попыток выполнения запроса (по умолчанию 5).
Возвращаемое значение:
True, если запрос был успешно выполнен, False в случае ошибки или превышения количества попыток.
Особенности:
Повторные попытки выполняются в случае, если база данных заблокирована.
Каждая попытка сопровождается задержкой перед следующей попыткой.
Пример использования:
python
Копировать код
execute_query_with_retry(conn, 'UPDATE users SET age = ? WHERE name = ?', (31, 'John'))
Применение:
Этот модуль предоставляет основные функции для взаимодействия с базой данных SQLite в рамках вашего проекта.
Он включает в себя обработку ошибок, логирование запросов и надежную обработку ситуаций с блокировкой базы данных.

Рекомендации:
Закрытие соединений: Убедитесь, что соединения с базой данных всегда корректно закрываются после использования,
чтобы избежать утечек ресурсов.
Логирование: Логирование запросов и операций с базой данных упрощает процесс отладки и анализа работы системы.

Описание файла translations.py
Этот файл отвечает за хранение текстов и переводов на различные языки, используемые в интерфейсе Telegram-бота. Также он содержит функции для генерации клавиатур, используемых в боте.

Компоненты файла:
Импорт библиотек:

InlineKeyboardButton и InlineKeyboardMarkup: Импортированы из модуля telegram для создания клавиатур в Telegram-боте.
Словари переводов:

button_texts: Словарь, содержащий тексты кнопок на различных языках. Каждый ключ в словаре ('en', 'ru', 'es', 'fr', и т.д.) представляет язык, а значение — список текстов кнопок на этом языке.
translations: Основной словарь переводов, содержащий ключевые фразы и их переводы на различные языки. Эти переводы используются для отображения сообщений и информации пользователям на их родном языке.
Функции:

language_selection_keyboard():
Генерирует клавиатуру для выбора языка, используя кнопку для каждого поддерживаемого языка.
Возвращаемое значение:
Объект InlineKeyboardMarkup, который представляет собой клавиатуру с кнопками для выбора языка.
Пример использования:
python
Копировать код
keyboard = language_selection_keyboard()
Переводы:
Каждый язык в словаре translations содержит следующие ключевые фразы:

order_confirmed: Подтверждение заказа.
proforma_number: Номер проформы.
event_date: Дата мероприятия.
time: Время.
people_count: Количество людей.
event_style: Стиль мероприятия.
city: Город.
amount_to_pay: Сумма к оплате.
status: Статус.
delivery_info: Информация о доставке.
currency: Валюта.
whatsapp_message: Сообщение для WhatsApp.
whatsapp_footer: Завершающий текст для сообщения в WhatsApp.
Эти переводы используются для формирования сообщений и кнопок интерфейса на разных языках, обеспечивая многоязыковую поддержку в боте.

Применение:
Этот модуль используется для работы с многоязыковыми текстами и генерации соответствующих клавиатур, что делает 
Telegram-бот доступным для пользователей, говорящих на различных языках.

Рекомендации:
Добавление новых языков: Чтобы добавить новый язык, достаточно добавить новый ключ с переводами в словари button_texts и translations.
Обновление переводов: Убедитесь, что переводы обновляются и проверяются на соответствие при добавлении новых фраз
или изменении существующих текстов в интерфейсе бота.
